Syntax

C++ syntax is very similer to the syntax in C

Core Program Structure:

When we consider a C++ program, it can be defined as a collection of objects that communicate with eachother by calling each other's methods.

  • Objects: These have states and behaviour. Example: A dog might be a specific breed and color
  • Class's: These contain a group of Objects and Methods
  • Methods: These are functions which are used to complete tasks
  • Variables: Values can be stored, changed and accessed using variables

Base Structure:

Let us look at some simple code that will print Hello World to the console
int main() {
cout << "Hello World"; // prints Hello World to the console
return 0;
}

Semi-colons and block scoping

Like in C, in C++ the semicolon is used to terminate a statement. Which means that each individual statement must be ended with a semicolon. The compiler will throw an error if you forgot one!!

x = y; // This one will complete first
y = y++; // This one will only run once the first statement has finished
Note: This means multiple statements can be on a single line, as long as there is a semicolon seperating them.
Scoping is the term used to describe the region that variables that have been declared can be accessed. There are two main types of variables, local variables and global variables.
Local variables that are declared inside of a funciton or other block are local variables. These can only be referenced in the current block or a child block.
Global variables are defined outside of all functions, usually at the top of the class. These variables can be referenced anywhere in the program.
Example:
int x = 23; // Global variable decleration
int main() {
int g = 200; // Local variable decleration
}

Comments

Comments are used to help anyone that is reading the program source code. C++ supports both single-line and block comments. Every character inside a comment is ignored by the compiler.
Even code you enter will be ignored!

// This is a single line comment

/**
This is a block comment, anything inside the comment block will be ignored
**/
